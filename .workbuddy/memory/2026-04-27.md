# 2026-04-27 工作日志

## SalesPilot CRM 项目优化（V5）

完成了对 `c:\Users\Administrator\Desktop\4月重点开发\V1\V5` 项目的全面安全与性能优化，涵盖 7 项改动：

### 已完成优化

1. **【安全-高优】密码 bcrypt 哈希** `backend/app/core/security.py`
   - 原来 `hash_password()` 直接返回明文，改为 bcrypt（rounds=12）真正加密
   - `verify_password()` 兼容旧明文密码（过渡期），新密码用 bcrypt 校验

2. **【安全】消除硬编码弱密码** `backend/main.py` + `backend/app/core/config.py`
   - 启动时 admin 初始密码从 `settings.ADMIN_DEFAULT_PASSWORD` 读取（默认 `Admin@2024#CRM`）
   - config 新增 `ADMIN_DEFAULT_PASSWORD` 和 `PASSWORD_MIN_LENGTH` 配置项

3. **【性能】analytics 改为 SQL 聚合** `backend/app/api/routes/analytics.py`
   - 原版全量加载所有商机/线索到 Python 内存做统计，大数据量下极慢
   - 重写为 SQL GROUP BY + SUM + COUNT 聚合，近期动态/客户漏斗限量查询

4. **【代码质量】Unicode 转义替换** `backend/app/services/crm_rules_service.py`
   - 阶段常量从 `\u521d\u6b65\u63a5\u89e6` 改为可读中文 `初步接触` 等

5. **【安全/性能】leads 列表增加分页** `backend/app/api/routes/leads.py`
   - 原来返回全量 list，改为 `PaginatedResponse`，默认 page_size=20，最大 200
   - 增加完整注释和类型提示

6. **【安全】密码强度校验** `backend/app/services/auth_service.py`
   - `validate_password()` 从仅"非空"改为最少 8 字符 + 至少一字母 + 至少一数字
   - `validate_username()` 增加正则校验（字母/数字/中文/下划线/连字符，2-50位）

7. **【体验】前端统一 Toast 提示** `frontend/crm-common.js`
   - 新增 `showToast(msg, type, duration)` 函数（info/success/warning/error）
   - 401 时不再静默跳转，先显示 "登录已过期…" Toast 并等待 1.8s 再跳转
   - 403/422/500 各有专属提示文案
   - `requireAuth()` 的 alert 改为 Toast
   - 网络层错误统一捕获并友好提示

### 依赖变更
- `requirements.txt` 新增 `bcrypt==4.1.3`（`passlib[bcrypt]` 已有但 security.py 直接调用 bcrypt）

### 注意事项
- 已有 admin 用户密码若是旧明文格式，verify_password 已做兼容（会走明文比较），建议登录后立即修改密码以触发 bcrypt 重新哈希
- 生产环境 `.env` 中务必设置 `ADMIN_DEFAULT_PASSWORD` 为强密码

## Bug 修复：page-leads.html `.map is not a function`

**问题**：`page-leads.html` 第 978 行 `(response || []).map(normalizeLead)` 报错，因为后端 `/api/leads` 已被改为分页接口，返回 `{ total, page, page_size, data: [...] }` 对象而非数组。

**修复**：参照 `page-opportunities.html` 的分页拉取模式，改为循环拉取所有页（每页最多 200 条），用 `response.data` 取数组再拼接，最后再 `.map(normalizeLead)`。
