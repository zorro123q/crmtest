手机和网页联调指南 Debug Guide

1. 启动本地前端静态服务

当前前端目录自带 serve.js，默认端口是 3000。如果要让手机访问，服务绑定地址要从 127.0.0.1 改成 0.0.0.0。

$env:HOST='0.0.0.0'
$env:PORT='3000'
npm run dev

2. 电脑浏览器联调

电脑本机直接访问下面地址即可打开本页：

http://127.0.0.1:3000/page-metadata.html

本页属于静态展示页，只看页面效果时不依赖后端接口。

3. 手机浏览器联调

手机和电脑接入同一局域网后，在电脑执行 ipconfig，找到 IPv4 地址，例如 192.168.1.23，然后在手机浏览器打开：

http://192.168.1.23:3000/page-metadata.html

4. 涉及接口时的地址设置

项目公共脚本 crm-common.js 在局域网访问时，会默认把接口基地址当成当前页面域名。若后端实际运行在 8000 端口，先在浏览器控制台执行：

localStorage.setItem('crm_api_base', 'http://192.168.1.23:8000')

然后刷新页面，再调登录、列表或提交类接口。

5. 后端跨域配置

后端当前默认只允许 localhost 和 127.0.0.1 来源。手机从 http://192.168.1.23:3000 打开页面时，需要把这个来源加入后端的 CORS_ORIGINS。

["http://localhost:3000","http://127.0.0.1:3000","http://192.168.1.23:3000"]

6. 推荐联调顺序

  1. 先在电脑浏览器确认静态页能正常打开。
  2. 再用手机访问同一页面，检查字号、滚动、点击区域和布局换行。
  3. 需要调接口时，再补充 crm_api_base 和后端 CORS。
  4. 排查手机问题时，用 Chrome 远程调试或手机浏览器开发者工具查看 Console 和 Network。
只预览 page-metadata.html 时，启动静态服务就够了。只有切到登录页、列表页或录入页这类接口页面时，才需要处理后端地址和跨域。
